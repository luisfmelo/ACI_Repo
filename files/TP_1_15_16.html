Trabalho Prático Nº1

Pilha protocolar Modbus TCP para clientes e servidores TCP

Introdução

Modbus é um protocolo de comunicações muito utilizado no meio industrial que permite a supervisão e o controlo remoto dos respetivos processos. Entre outras possibilidades, o protocolo Modbus pode funcionar sobre a Internet (ou uma intranet) utilizando TCP/IP, o que lhe confere uma enorme flexibilidade em termos de ligação dos equipamentos em causa. Neste trabalho pretende-se desenvolver uma pilha protocolar Modbus TCP sobre uma interface de sockets TCP/IP, constituíndo uma biblioteca de funções necessárias à comunicação com Modbus. O trabalho será realizado em grupos de 2 elementos (da mesma turma) e deverá ser concluído até 18 de Março.

Objectivos

Os objectivos principais do trabalho são:

  • Compreender os conceitos de camada, serviço e protocolo em redes industriais.
  • Concretizar os conceitos anteriores através da implementação de uma API para desenvolver clientes e servidores Modbus TCP, usando sockets..
  • Aprofundar o conhecimento do protocolo Modbus.

Ambiente de Desenvolvimento

O trabalho terá que ser desenvolvido em linguagem C capaz de ser compilado num ambiente gcc/Linux. Para aqueles que preferem trabalhar sobre o Windows recomenda-se o Cygwin (já instalado no laboratório). Este software disponibiliza um ambiente de desenvolvimento que emula o Linux sob uma plataforma Windows.

Documentação

Para realizar este trabalho é necessário ler previamente a documentação relativa ao protocolo Modbus. Sugere-se a seguinte ordem na leitura dos documentos:

  1. Modbus chapter - tem uma descrição completa do funcionamento e do protocolo Modbus. No entanto, os detalhes relativos à implementação são descritos nos doscumentos seguintes.
  2. Modbus Application Protocol Specification - descrição detalhada do modelo de dados e das funções. Devem ser lidos os capítulos 3-5 e o capitulo 7 (Exceptions).
  3. Modbus Messaging on TCP/IP Implementation Guide - descrição da implementação do protocolo sobre TCP/IP. Devem ser lidos os capítulos 1-3. O capitulo 4 deve ser lido numa fase mais avançada do trabalho.

Requisitos

Suportar o desenvolvimento de clientes e servidores Modbus TCP. Em particular, os clientes deverão suportar as seguintes funções Modbus:

  • 01 (0x01) Read Coils
  • 15 (0x0F) Write Multiple Coils

A pilha protocolar Modbus TCP deverá ser organizada em duas camadas, nomeadamente de aplicação e de sessão. Dever-se-á respeitar a forma de comunicação entre camadas típica de interações cliente-servidor com Request-Confirmation-Indication-Response. As funções usadas em cada camada deverão ser guardadas em ficheiros separados, nomeadamente ModbusAP.c e ModbusTCP.c e as funções header (.h) correspondentes.

Implementação

A implementação da pilha Modbus TCP assentará sobre a interface de programação Sockets. Esta interface já foi estudada previamente (Sistemas de Informação). No entanto recomenda-se fortemente a leitura do seguinte livro que descreve de forma clara o desenvolvimento de clientes e servidores utilizando sockets. Possui muitos exemplos em C 'prontos a usar'. O livro está disponível na Biblioteca (onde existem vários exemplares)

    • D. Comer, Internetworking with TCP/IP, Volume 3: Client-Server Programming and Applications

Para testar o trabalho recomendam-se as seguintes aplicações (Clientes / Servidores / Mestres / Escravos Modbus):

  • Livres
    • Digslave Modbus Slave Simulator - servidor Modbus (escravo). Invocação via linha de comando.
    • Modpoll Modbus Master Simulator - cliente Modbus (mestre). Invocação via linha de comando.
  • Limitados (funcionam apenas durante 21 dias)
    • Modbus Slave - servidor Modbus (escravo). Gráfico(ambiente Windows).
    • Modbus Poll - cliente Modbus (mestre). Gráfico(ambiente Windows).
